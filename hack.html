UT GRAND CHALLENGE HACKATHON: Teaching Autonomous Vehicles to Behave

 

Time: 
8 hours
Team:
Ritambhara Singh - Lead Designer and Presenter
Peidi Sun - Designer
Yajie Niu - Developer
Kevin Wang - Developer
Tools:
Sketch - Design and Prototype
Android Studio - App Development


Problem Space
To explore how humans can help autonomous vehicles learn social norms. What decisions are these cars making — such as stopping (or not stopping) when a pedestrian is crossing the road, or how an autonomous car behaves to avoid a collision — and do drivers think they're ethical? To improve future technologies and guide autonomous vehicles so they make decisions we all agree are safe, acceptable, and permissible.


To design an easy-to-use interface so that the drivers can rate their autonomous vehicles.


Process


Brainstorming:


As a team, we brainstormed to find out the challenges a driver may face to give a car a positive and negative feedback after it performs any action.The challenges we listed:

  1. What is the best possible way to give feedback?
  2. How to optimize the number of responses given by the driver?

Also, we realized that it is important to give a reason to the car for every feedback, so that the car can learn if a particular action is suitable in a particular context.



Raw notes and sketches


System Concept Generation

Assumption: All the algorithms and technologies work with 100% accuracy.

The challenge was to design a user interface for drivers through which they can give feedback for vehicle's action and which can be fed to the machine learning algorithm. Our main aim was to take user's feedback in the least intrusive way possible. This was to ensure that the user doesn't get frustrated while giving feedback. This would maximize the number of feedback and help the vehicle learn even more.


How does the learning algorithm work?
The learning algorithm that was given as a part of the challenge learned through evaluative reinforcement. On every action, a manual feedback was given to the agent. The agent will keep on learning through these feedbacks. The evaluative responses can be positive or negative.
How does our system work?
The system will keep receiving user feedback through different sources. It will associate the feedback (positive or negative) with the action taken by the vehicle and help it learn the 'preferences' of its driver.


Our system comprised of the following elements:

1. Voice Recognition Model :
This would take input of the driver's verbal comments. This would also take into account user's involuntary verbal expressions and interpret them as positive or negative.


2. Facial Recognition Model :
This would take input of the driver's facial expressions and interpret them as positive or negative.


3. Wearable Devices :
Input from wearable devices like smart watches can provide an insight on how the body of the driver reacts to a situation. These reactions can be interpreted. For example- if in a situation the heart rate of the driver increases rapidly, it can be interpreted as a negative feedback.


4. Voluntary User Feedback:
The driver can provide a feedback voluntarily through a mobile application. The UI design of this application is the main focus of the rest of this project.


Fedbacks from above sources will be fed to the machine learning algorithm. Based on them, the algorithm and hence the vehicle will learn the actions preferred by the user. Eventually, it will learn the social norms of that society.
